# Plan: Инициализация проекта Tasker

## Цель

Создать минималистичное TUI-приложение для управления задачами на Rust.
Работает на macOS и Windows. Данные хранятся в `tasks.json` рядом с запуском.

---

## Шаг 1 — Инициализация Rust-проекта

- [ ] `cargo init` в текущей директории
- [ ] Настроить `Cargo.toml`: имя, версия, edition = "2021"
- [ ] Добавить зависимости:
  - `ratatui` — TUI-фреймворк
  - `crossterm` — кросс-платформенный терминальный бэкенд
  - `tui-textarea` — многострочный редактор
  - `serde` + `serde_json` — сериализация
  - `chrono` — работа с датами (created_at, подсчёт дней)
  - `arboard` — чтение HTML из буфера обмена (кросс-платформенно)
  - `anyhow` — обработка ошибок

---

## Шаг 2 — Структура файлов

Создать следующую структуру `src/`:

```
src/
├── main.rs      # Точка входа, инициализация терминала, главный цикл
├── app.rs       # Состояние приложения и бизнес-логика
├── ui.rs        # Отрисовка интерфейса
├── input.rs     # Обработка нажатий клавиш
├── storage.rs   # Чтение и запись tasks.json
└── links.rs     # Парсер ссылок и умная вставка из HTML
```

---

## Шаг 3 — Модель данных (`app.rs`)

Определить структуры:

```rust
struct Task {
    id: u64,
    text: String,           // многострочный текст
    in_progress: bool,
    created_at: NaiveDate,  // дата создания (chrono)
}

struct App {
    tasks: Vec<Task>,
    selected: usize,
    mode: Mode,
}

enum Mode {
    Normal,
    Add,
    Edit,
}
```

Реализовать методы `App`:
- `add_task(text)` — добавить задачу в конец
- `delete_task(index)` — удалить задачу
- `toggle_in_progress(index)` — переключить флаг
- `edit_task(index, text)` — обновить текст

---

## Шаг 4 — Хранилище (`storage.rs`)

- `load(path) -> Vec<Task>` — загрузить из JSON, вернуть пустой список если файл не существует
- `save(path, tasks)` — сохранить в JSON
- Путь: `std::env::current_dir()` + `tasks.json`
- Сохранение вызывается сразу после каждого изменения

## Шаг 5 — Инициализация терминала (`main.rs`)

- Включить raw mode через crossterm
- Включить захват событий мыши (`EnableMouseCapture`)
- Создать альтернативный экранный буфер (чтобы не засорять историю терминала)
- Запустить главный цикл: обработка событий → обновление состояния → отрисовка
- При выходе: отключить захват мыши, восстановить терминал в исходное состояние

## Шаг 6 — Отрисовка интерфейса (`ui.rs`)

Три компонента:

1. **Список задач** — основная область экрана, каждая задача — отдельная карточка
   - Карточка: рамка + заголовок с текстом задачи слева и датой/днями справа + тело
   - Обычная карточка: `BorderType::Plain`, цвет по умолчанию
   - "В процессе": `BorderType::Thick`, цвет по умолчанию
   - Выбранная: `BorderType::Plain`, рамка оранжевым (`Color::Rgb(255, 165, 0)`)
   - Выбранная + "в процессе": `BorderType::Thick`, рамка оранжевым
   - Если задача висит > 30 дней — дата и кол-во дней в заголовке подсвечиваются красным
   - Многострочный текст отображается внутри карточки
   - Ссылки рендерятся в footnote-стиле: `текст[N]` в теле + `[N] url` внизу карточки
   - Каждая пара текст+url получает свой цвет из палитры (циклически):
     1. Голубой `(100, 210, 220)`
     2. Лаванда `(180, 140, 230)`
     3. Мятный `(100, 220, 160)`
     4. Розовый `(230, 140, 180)`
     5. Жёлто-зелёный `(190, 220, 100)`
     6. Персиковый `(230, 180, 120)`
   - Скроллбар справа от списка (виджет `Scrollbar` из ratatui)

2. **Редактор** (режимы Add / Edit) — всплывающий блок поверх списка
   - Использует `tui-textarea`
   - Заголовок: "Новая задача" или "Редактировать"

3. **Строка подсказок** — зафиксирована внизу
   - В режиме Normal: `[a] добавить  [e] изменить  [d] удалить  [space] в процессе  [q] выйти`
   - В режимах Add/Edit: `[Ctrl+S] сохранить  [Esc] отменить`

## Шаг 7 — Обработка ввода (`input.rs`)

### Режим Normal — клавиши

| Клавиша | Действие |
|---|---|
| `↑` / `k` | курсор вверх |
| `↓` / `j` | курсор вниз |
| `a` | перейти в режим Add |
| `e` | перейти в режим Edit |
| `Space` | переключить "в процессе" |
| `d` | удалить задачу |
| `q` | выйти |

### Режим Normal — мышь

| Событие | Действие |
|---|---|
| Клик по карточке | выбрать задачу |
| Колёсико вверх/вниз | прокрутка списка |

### Режим Add / Edit

| Клавиша | Действие |
|---|---|
| `Ctrl+V` | умная вставка: HTML → `[текст](url)`, иначе обычный текст |
| `Ctrl+S` | сохранить, вернуться в Normal |
| `Esc` | отменить, вернуться в Normal |
| остальные | передать в tui-textarea |

## Шаг 8 — Парсер ссылок (`links.rs`)

Отдельный модуль для работы со ссылками в тексте задачи:

- `parse_links(text) -> Vec<(range, text, url)>` — найти все `[текст](url)` в строке
- `render_with_footnotes(text) -> (display_text, footnotes)` — преобразовать для отображения
- `html_to_markdown(html) -> String` — извлечь `<a href="...">текст</a>` из HTML и заменить на `[текст](url)`, остальное — plain text
